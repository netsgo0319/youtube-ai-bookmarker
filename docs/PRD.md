# 📘 **PRD — AI 기반 YouTube Second Brain (MVP)**

**작성 기준**

* 아이디어 A(지식노동자용 유튜브 세컨드 브레인)
* 우선순위 1: 영상 → 텍스트 → 문장/클립 검색 → 타임스탬프 점프
* Value Positioning:

  > 우리 제품은 유튜브와 각종 영상으로 정보를 수집하는 지식 노동자와 팀이 예전에 봐둔 ‘그 설명·코드·문장’을 다시 찾고자 할 때 북마크, 노션, 수동 타임스탬프, 유튜브 검색을 뒤지는 기존 방식보다 **모든 영상을 자동 텍스트화·클립화하여 문장 검색 → 즉시 타임스탬프 이동이 가능하게 함으로써 인사이트 재발견·재사용을 빠르고 정확하게 만드는 제품**이다.

# 1. 🎯 **제품 목표 (Product Vision)**

지식노동자, 전문가, 강사, 연구형 크리에이터 등이
**영상에서 얻은 인사이트를 바로 찾고, 바로 쓰고, 바로 재사용할 수 있도록**
유튜브 영상을 **텍스트 자료처럼 검색·정리·활용** 가능하게 만드는
**AI 기반 개인/팀용 영상 지식베이스**.

# 3. 🥇 **핵심 문제 정의 (Problem Statement)**

### 현재 모든 사용자는 공통적으로:

1. 영상 자료가 여러 곳에 흩어져 재생산 불가
2. “예전에 본 그 장면” 검색 불가
3. 폴더링/태그/정리 자동화는 부정확
4. 타임스탬프·클립 단위 관리는 수작업
5. 영상은 팀 문맥이 반영되지 않음

➡ **영상 인사이트를 저장해도 다시 찾지 못하는 게 본질적 문제**

---

# 4. ⭐ **해결해야 할 핵심 니즈 (Top Needs)**

1. 영상 자동 텍스트화
2. 문장/키워드/코드 검색
3. 검색 결과에서 바로 타임스탬프로 이동
4. 자동 요약 + 클립 자동 분해
5. 프로젝트/폴더 자동 분류
6. 기존 지식 구조(Notion/Obsidian 태그와 유사 구조) 지원

**MVP에서는 1~3번을 최우선 구현.**

---

# 5. 🧱 **제품 범위 (MVP Scope)**

MVP는 아래 4가지만 구현한다.

---

## ✔️ **MVP v1 기능 목록**

### **1) YouTube 링크 입력 → 영상 자동 처리**

* 사용자가 링크 입력하면 서버에서 영상 메타데이터 획득
* 영상 자막/음성을 OpenAI로 텍스트화
* 텍스트를 한 문장 단위로 분리하여 저장
* Supabase DB에 저장

---

### **2) 문장/키워드/코드 검색 엔진**

* 텍스트 기반 full-text 검색
* 결과 클릭 → 해당 타임스탬프(초단위)로 점프
* UI는 “문장 snippet + 시간 정보”로 출력

---

### **3) 기본 요약 + 자동 클립 후보 생성**

* OpenAI로 5~10개 주요 요약 생성
* 텍스트 기준으로 “주제 전환점”을 클립 후보로 자동 감지
* 클립 클릭 → 영상 특정 지점으로 이동
* 클립은 태그 없이 “클립1, 클립2…”로만 구성(단순 MVP)

---

### **4) 자동 Inbox + 기본 폴더링**

* 모든 영상은 기본 Inbox에 들어감
* 사용자가 폴더를 만들고 드래그해서 이동 가능
* 최초에는 자동 분류 알고리즘 없음

  * 단, 제목 기반 “기본 제안 폴더”만 제공 (예: "React", "Marketing")

---

# 7. 🖼 **핵심 사용자 시나리오 (User Journey)**

## **시나리오 1: 지식노동자 박지훈**

1. React Suspense 설명 영상을 봤지만 나중에 찾기 어려움
2. 우리 서비스에 링크 붙여넣음
3. 2분 뒤 전체 텍스트화 완료
4. “fallback” 검색 → 바로 해당 타임스탬프 이동
5. 폴더에 "React"로 정리
   ➡ *기존 대비 재검색 시간 15분 → 10초*

---

## **시나리오 2: UX 디자이너 하람**

1. 영상 5개를 참고해 와이어프레임 작업
2. “왜 이 화면 구조가 중요하다” 라는 문장을 다시 찾고 싶음
3. 검색창에 “왜 중요” 입력
4. 자동 감지된 3개 문장 → 클릭 시 즉시 영상 이동
   ➡ *영감 재활용 가능*

---

## **시나리오 3: 크리에이터 윤서**

1. TED 영상에서 10개 인사이트를 찾음
2. 클립 자동 후보를 클릭하여 빠르게 검토
3. 스크립트 작성에 필요한 부분만 타임스탬프 저장
   ➡ *팀 편집 속도 증가*

# 8. 📐 **기능 요구사항 (Functional Requirements)**

---

## **FR1. YouTube 링크 처리**

* 사용자는 URL을 입력할 수 있다
* 영상 메타데이터(title, channel, thumbnail) 저장
* 자막이 있으면 수집 / 없으면 OpenAI로 STT 처리
* DB(Supabase)에 영상·텍스트·타임스탬프 저장

---

## **FR2. 텍스트 검색 (핵심)**

* 사용자는 검색창에 키워드를 입력
* 시스템은 문장 기준으로 search 결과 반환
* 문장 클릭 → 해당 초(second) 위치로 유튜브 플레이어 이동
* 검색은 최소 200자까지 빠르게 처리해야 함

---

## **FR3. 자동 요약**

* OpenAI API로 요약 요청
* 짧은 요약(3점) + 긴 요약(7점) 제공
* 요약은 한 번 요청하면 캐싱

---

## **FR4. 자동 클립 후보**

* 텍스트 기반 주제 전환점 감지
* 후보 클립 목록을 타임스탬프 기준으로 표시
* 클릭 → 재생 위치 이동

---

## **FR5. 폴더·Inbox 관리**

* Inbox: 모든 신규 영상 자동 저장
* 사용자는 폴더 생성 가능
* drag & drop으로 이동
* 기본 추천 폴더: 제목 내 주요 키워드 기반
